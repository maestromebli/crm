# User stories: воркспейс угоди

Формат: **Як … хочу … щоб …**

## Готовність до виробництва

1. Як **менеджер з продажу**, хочу бачити чек-лист готовності на бічній панелі, щоб зрозуміти, що блокує передачу у виробництво.
2. Як **менеджер**, хочу щоб після змін оплати, заміру, договору або файлів система **зберігала знімок** готовності, щоб мати історію для аудиту та AI.
3. Як **керівник**, хочу викликати `GET /api/deals/:id/readiness-history`, щоб переглядати останні знімки без залізної прив’язки до UI.

## Договір і метадані

4. Як **менеджер**, хочу оновлювати метадані воркспейсу з вкладок (оплата, замір, кваліфікація), щоб readiness автоматично перераховувався.
5. Як **менеджер**, хочу змінювати статус договору, щоб пункт «договір підписано» у готовності відображав фактичний стан.

## Файли та версії

6. Як **менеджер**, хочу додати перший файл за URL, щоб з’явився логічний `FileAsset` і можна було додавати **нові версії** тим самим `fileAssetId`.
7. Як **менеджер**, хочу щоб креслення або лист заміру зараховувались у readiness, щоб не відправляти пакет без технічної бази.

## Передача (handoff)

8. Як **менеджер**, хочу відправити пакет на прийняття (`DealHandoff` → SUBMITTED), щоб виробництво бачило запит у системі.
9. Як **виробництво**, хочу прийняти або відхилити пакет з причиною, щоб історія залишилась у `ActivityLog`.
10. Як **менеджер**, хочу лишати нотатки передачі окремо від загальних метаданих, щоб не змішувати контекст з `workspaceMeta`.

## Автоматизація (майбутнє)

11. Як **адмін**, хочу створювати правила з тригером на подію воркспейсу, щоб пізніше запускати сценарії без зміни коду (зараз — лише запис `AutomationRun`).
