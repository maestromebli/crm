# Вхід адміністратора

## Основний обліковий запис (повний доступ)

| Поле     | Значення           |
|----------|--------------------|
| Email    | `admin@enver.com`  |
| Пароль   | `admin123`         |
| Роль у БД | `SUPER_ADMIN`     |
| Права    | Усі записи з `PermissionKey` (після `pnpm db:seed`) |

Після входу відкривається дашборд. Якщо сесія вже активна, відкриття `/login` перенаправляє на `/dashboard`.

## Підготовка БД

```bash
pnpm db:push
pnpm db:seed
```

Або лише адміністратора (швидше, якщо seed уже частково був):

```bash
pnpm db:ensure-admin
```

Без запису користувача `admin@enver.com` у таблиці `User` вхід завжди дасть помилку «Невірний email або пароль».

У **терміналі сервера** (`pnpm dev`) у режимі development додатково зʼявиться лог `[auth]`, якщо користувача не знайдено або БД недоступна.

## Змінні середовища

У `.env.local` потрібні `DATABASE_URL`, `NEXTAUTH_URL`, `NEXTAUTH_SECRET` (див. `.env.example`).
